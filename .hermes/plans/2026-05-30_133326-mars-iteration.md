# Mars Iteration Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Add Mars as the next single body in the physics-first C/raylib solar-system simulator, preserving the existing Sun + Mercury + Venus + Earth + Moon behavior and rendering/trail features.

**Architecture:** Keep the simulation truth in `src/sim/` as double-precision SI values and add Mars through explicit constants, a Mars body factory, and a new scene factory. Keep all raylib-specific color, radius, position conversion, trails, overlay text, and app selection in `src/render/` and `src/main.c`. Do not add Phobos/Deimos, Mars axial tilt, rotation, ephemerides, or barycentric corrections in this iteration.

**Tech Stack:** C11, raylib 6.0, Makefile, headless C test binaries, velocity-Verlet N-body integration.

## Source Data for Mars

Use the same style as existing constants: simple SI constants and deterministic perihelion initialization.

Authoritative facts used for this milestone:

- NASA Science Mars facts: Mars radius is about `3,390 km`, average distance about `228 million km`, year about `687 Earth days`.
- NSSDC-derived Mars/Earth comparison table:
  - mass: `0.6419 × 10^24 kg`
  - volumetric mean radius: `3390 km`
  - semimajor axis: `227.9 × 10^6 km`
  - eccentricity: `0.0934`
  - perihelion: `206.6 × 10^6 km`
  - aphelion: `249.2 × 10^6 km`
  - sidereal orbit period: `686.980 days`

Constants to add:

```c
#define SOLAR_MARS_MASS_KG 6.419e23
#define SOLAR_MARS_RADIUS_M 3390000.0
#define SOLAR_MARS_SEMI_MAJOR_AXIS_M 227900000000.0
#define SOLAR_MARS_ECCENTRICITY 0.0934
#define SOLAR_MARS_PERIHELION_M \
    (SOLAR_MARS_SEMI_MAJOR_AXIS_M * (1.0 - SOLAR_MARS_ECCENTRICITY))
#define SOLAR_MARS_APHELION_M \
    (SOLAR_MARS_SEMI_MAJOR_AXIS_M * (1.0 + SOLAR_MARS_ECCENTRICITY))
#define SOLAR_MARS_PERIHELION_SPEED_MPS \
    (sqrt(SOLAR_G * SOLAR_SUN_MASS_KG * \
        ((2.0 / SOLAR_MARS_PERIHELION_M) - (1.0 / SOLAR_MARS_SEMI_MAJOR_AXIS_M))))
```

Derived values with current simulator `G` and Sun mass:

- perihelion distance: `206614140000.0 m`
- aphelion distance: `249185860000.0 m`
- perihelion speed: `26501.588011990192 m/s`
- Kepler period from current constants: about `686.784097062 days`

The plan should not require the derived period to exactly match the rounded NSSDC `686.980 days`, because the semimajor axis and solar GM are rounded differently. Tests should assert formulas/behavior, not snapshot exact period literals.

## Proposed Initial Mars State

Use a new axis not occupied by the current planet perihelion placements to keep the startup scene readable.

Recommended Mars perihelion state:

```c
Body solar_system_create_mars_at_perihelion(void)
{
    return body_create(
        "Mars",
        BODY_KIND_PLANET,
        SOLAR_MARS_MASS_KG,
        SOLAR_MARS_RADIUS_M,
        (Vec3d){0.0, 0.0, -SOLAR_MARS_PERIHELION_M},
        (Vec3d){SOLAR_MARS_PERIHELION_SPEED_MPS, 0.0, 0.0},
        false
    );
}
```

Reasoning:

- Position at `-Z`, velocity `+X` gives `r × v = -Y`, matching the established prograde sign.
- Mars is visually separated from Earth at startup, which is useful with the illustrative rendering and trail features.
- This is deterministic perihelion initialization, not an attempt at real current ephemeris.

static void test_mars_roughly_returns_after_one_orbit(void)
{
    SolarSystem system = solar_system_create_sun_mercury_venus_earth_moon_mars();
    Vec3d initial = system.bodies[5].position_m;
    const double dt_seconds = SOLAR_DAY_SECONDS;
    const double mars_period_seconds = 2.0 * acos(-1.0) * sqrt(
        (SOLAR_MARS_SEMI_MAJOR_AXIS_M * SOLAR_MARS_SEMI_MAJOR_AXIS_M * SOLAR_MARS_SEMI_MAJOR_AXIS_M) /
        (SOLAR_G * SOLAR_SUN_MASS_KG)

    const int steps = (int)(mars_period_seconds / dt_seconds);

    for (int i = 0; i < steps; ++i) {
        solar_system_step(&system, dt_seconds);
    }

    double distance_from_initial = vec3d_length(vec3d_sub(system.bodies[5].position_m, initial));
    assert(distance_from_initial < 0.12 * SOLAR_AU_METERS);
}

## Risks and Caveats

- The fixed Sun baseline remains an approximation. Adding Mars increases N-body interactions but does not make the model ephemeris-accurate.
- The Moon’s Earth-relative stability should not be invalidated by Mars, but implementation should re-run existing lunar boundedness tests and be ready to adjust only if evidence shows timestep instability.
- Mars orbital period from rounded constants will not exactly equal the public fact-sheet value. Use formula-based tests and loose orbit-return bounds.
- Real orbital inclinations are still ignored. Mars is placed in the simulator’s current XZ orbital plane for deterministic readability.
- Mars has moons, but Phobos/Deimos are out of scope for this iteration.
